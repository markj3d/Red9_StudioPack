Pro_Pack : exporter¶

ProPack exporter module is our own export management system for handling the output of files based on the metaData export node class systems. This module is largely the UI management as the majority of the codebase is bound to the metadata_pro mNode classes

>>> # import statement for the module via the r9pro decompiler
>>> from Red9.pro_pack import r9pro
>>> r9exp = r9pro.r9import('r9exp')

class ExportManager¶

Bases: object

Main Export handler used by all functions, this expects the scenes to have valid ExportTags and ExportLoops setup for this process to pickup & relies purely on the Red9 MetaData systems so all rigs must be correctly setup in the RigManager before running this codebase.

There are a few key class variables that dictate what output processes the run() and run_files functions call:

  • self.output_fbx : export fbx using the “selective” mode, soon to be merged so we only have one call for FBX
  • self.output_r9anim : save an r9Anim file
  • self.output_export : main export call for FBX / Unreal / Unity animation data
  • self.output_mayafile : save the current scene as named - used by some of the internal processes
  • self.output_playbast : create a Maya playblast for each loop, settings controlled by the Playblast Settings UI
  • self.output_mayaloops : save a Maya file per export Loop named as per the loop data
  • self.export_tag* : optional specific ExportTags (mNodes) for processing, if not set we process all ExportTags found in the scene
  • self.export_loops : optional specific ExportLoops (mNodes) for processing, if not set we process all ExportLoops found in the scene
  • self.output_abc : optional export to Alembic

Note

that if self.export_tags is given (list of mNodes already instantiated) then we only process ExportLoops connected to those tags. This allows you to specifically output just a given character from a list iof scenes if required.

>>> from Red9.pro_pack import r9pro
>>> r9exp = r9pro.r9import('r9exp')
>>> ex = r9exp.ExportManager()
>>> ex.output_export =True
>>> ex.run()

export_fbx(**kws)¶

run the fbx anim transfer export process

Parameters:
  • runSceneRoot – run the sceneRoot process during the export, default = False
  • mode –

    ‘simple’ (default) or ‘selection’ 2 different ways to produce the fbx data.

    • ’simple’ : creates a duplicate bound skeleton and exports that data, giving you just the base skeleton output, aimed at animation data
    • ’selection’ : exports the rigs nodes directly, including the skins and blends, whilst the data included will be

    minimum, skeleton, mesh etc, it will always include the rigs upper hierarchy in the data

  • respect_settings – do we respect and use the fbx_settings from the base UI or ignore them?
  • geoflags – default=False, if True we also include EmbeddedTextures, SmoothingGroups, Tangents in the fbx data else we assume that the fbx data is aimed at animation data only
  • bakeanim – True by default, use the bakeanim flag in the fbx output
