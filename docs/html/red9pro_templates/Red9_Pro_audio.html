Pro_Pack : Audio¶

ProPack audio module manages BWav formats, Timecode and the more advanced aspects of dealing with Audio within Maya. All the BWav handling in the StudioPack ultimately calls this codebase but most of the actual handling in ProPack still uses the AudioNode and AudioHandler from the StudioPack API.

>>> # import statement for the module via the r9pro decompiler
>>> from Red9.pro_pack import r9pro
>>> r9pro.r9import('r9paudio')
>>> import r9paudio

This module is extensions to the main audio support through the Red9 StudioPack and requires the StudioPack codebase to run

milliseconds_to_Timecode(milliseconds, smpte=True, framerate=None)¶

convert milliseconds into correctly formatted timecode

Parameters:
  • milliseconds – time in milliseconds
  • smpte – format the timecode HH:MM:SS:FF where FF is frames
  • framerate – when using smpte this is the framerate used in the FF block default (None) uses the current scenes framerate

Note

  • If smpte = False : the format will be HH:MM:SS:MSS = hours, minutes, seconds, milliseconds
  • If smpte = True : the format will be HH:MM:SS:FF = hours, minutes, seconds, frames

timecode_falls_within(timecode, timecode_bounds)¶

from a given list of timecode bounds in smpte format [(startTC, endTC),....] see if the given TC falls within any of them and if so return that timecode section. This is a little like searching an EDL for the correct take based on a given timecode

>>> timecode_falls_within('06:35:01:18', [('06:34:36:17', '06:34:40:11'),
>>>                                       ('06:34:59:21', '06:35:03:24'),
>>>                                       ('06:33:25:28', '06:33:31:10')]
>>> # Result : ('06:34:59:21', '06:35:03:24')

Alternatively if we have a node with timecode stamped we can do the following

>>> tc=r9Audio.Timecode('my_node_with_timecode')
>>> timecode_falls_within('06:35:01:18', tc.getTimecode_bounds(tc_only=True)))

getTimecode_bounds(tol=2, tc_only=False)¶

from node bound to the class find all timecode bounds and return as a list of tuples. This is designed to handle discontinuous timecode, ie a scene with multiple cuts that have multiple timecode slices.

Parameters:
  • tol – key gradient thats allowed, usually count attr is a linear counter of val 1 but default for this is 2 to give us a margin of error
  • tc_only – if True we only return the found bounds as timecode data [(startTC, endTC)....] else the return includes the frame times [(startFrm, endFrm, startTC, endTC)]

Note

timecode bounds in this case are obvious slices of timecode within an animation curve, a rig may have many different timecode sections bound to it and this will find and return those ranges for processing, think of them as cuts in an EDL edit
