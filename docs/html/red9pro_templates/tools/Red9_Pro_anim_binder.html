Pro_Pack : tools : anim_binder¶

ProPack anim_binder module is the replacement of the original AnimationBinder module which was done back in 2011 for the Autodesk MasterClass support files. This deals with creating the BND binder files themselves.

>>> # import statement for the module via the r9pro decompiler
>>> from Red9.pro_pack import r9pro
>>> r9anim_binder = r9pro.r9import('r9anim_binder')

Red9 Base Animation Binder systems designed to build your own BND files used by the Pro_to_BND setup in ProPack. This is essentially an upgraded version of the original codebase produced for the Autodesk MasterClass

bind_skeletons(source, dest, method='connect', manage_scales=False, verbose=False, unlock=False, bindroot=True, bindroot_inc=False, **kws)¶

From 2 given root joints search through each hierarchy for child joints, match them based on node name, then connect their transform channels trans/rots directly, or parentConstrain them depending on the method flag.

Parameters:
  • source – root of the source skeleton
  • dest – root of the destination skeleton
  • method – the method used for the connection, either ‘connect’ or ‘constrain’
  • manage_scales – bool, do we also look at binding jnt scales where applicable, this only runs if the source jnt has incoming scale connections which would then be propagated via a scaleConstrain
  • unlock – if True force unlock the required transform attrs on the destination skeleton first
  • bindroot – if True (default) we set a parentConstraint between the source and dest rootJoint, else they remain unbound
  • bindroot_inc – if bindroot=True then this is ignored. This flag forces the 2 root jnts into the main bind loop, they’re ignored by default if the bindroot flag is on as that forces a parentConstraint between them.

class BindNodeAim(source=None, destination=None, upVector=None, settings=None)¶

Bases: Red9.pro_pack.tools.anim_binder.BindNodeBase

Third more complex Bind type used for complex Spine issues. Unlike the other methods this uses an AimConstraint to point the controller towards a target node to extract a rotation vector. This gets over issues with complex spline setups where the previous 2 methods would send in incorrect rotation data.
